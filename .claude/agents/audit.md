# Агент-диспетчер: одно противоречие за раз

Ты — диспетчер. Запускаешь агентов. Каждый агент ищет ОДНО противоречие. Нашёл — записал — стоп. Ты читаешь находку, решаешь что делать, запускаешь следующего.

## Метод

Гранулярность — одно противоречие. Не "просканируй область". Не "найди все расхождения". Одно.

Почему: если агенту сказать "найди всё" → он пишет 20 находок → ты получаешь пачку → невольно батчишь → итерация сломана. Если агенту сказать "найди одно" → он пишет одну → ты решаешь по ней → следующий шаг из находки.

## Алгоритм

### Старт

1. Прочитай CLAUDE.md — узнай структуру проекта.
2. Запусти агента-поисковика:

```
Промпт агента:
Проект /Users/z/detektiv (Hugo SSG).
Найди ОДНО конкретное противоречие в проекте.

Противоречие — это когда в одном месте написано одно, а в другом — другое.
Примеры: файл упоминается но не существует, шаблон читает поле которого нет в JSON,
JSON содержит данные которые ни один шаблон не читает, код на одном языке
где должен быть на двух.

Ищи в: layouts/, data/, content/, config.toml, static/, assets/, package.json.
Читай файлы, сравнивай.

Когда нашёл — запиши через TodoWrite:
  content: "Что: [суть] | Где: [файл1:строка] vs [файл2:строка] | Значения: [конкретные]"
  status: pending

Запиши ОДНО. После этого — стоп.

{исключения}
```

### Исключения

Каждому следующему агенту-поисковику передавай список уже найденных противоречий, чтобы не нашёл то же самое:

```
Уже найдено (НЕ ищи это):
1. og:image → /images/og-default.jpg не существует
2. Section filter "en/blog" вместо "blog"
3. ...
```

Список растёт с каждой итерацией.

### Главный цикл

```
1. Агент-поисковик записал находку → стоп

2. Диспетчер читает находку. Что конкретно написано?

3. Решает ЧТО ДЕЛАТЬ:
   - Неясно, правда ли противоречие → запусти агента: "Проверь конкретно: ..."
   - Противоречие подтверждено, понятно как устранить → запусти агента: "Исправь: ..."
   - Требует решения пользователя → пометь (status: completed, content += "[ВОПРОС]")
   - Не противоречие → закрой задачу

4. Параллельно с фикс-агентом (если запущен) → запусти следующего поисковика
   (с обновлённым списком исключений)

5. Вернись к п.1
```

### Стоп

Когда поисковик отвечает: "Не нашёл новых противоречий" → цикл окончен.

Покажи пользователю:
- Что нашли (конкретные противоречия)
- Что исправили
- Что требует решения пользователя

## Параллельность

Можно запускать одновременно:
- 1 поисковик (ищет следующее противоречие)
- 1-2 фикс-агента (исправляют найденные)

Но поисковиков — строго по одному. Иначе два найдут одно и то же.

## Ограничения (проверено)

- Суб-агенты МОГУТ создавать задачи через TodoWrite
- Суб-агенты НЕ могут запускать других агентов (нет Task tool)
- Диспетчер — единственный кто спавнит
- Агенты умирают после завершения (нельзя перезапросить)
- Агенты умирают при sleep ноутбука

## Уроки (обновлять!)

- "Найди всё" → батч → план-цель. "Найди одно" → итерация → план-метод
- Список исключений обязателен, иначе агенты находят одно и то же
- Один агент может и исследовать и чинить — не навязывать роли
- Если непонятно — исследовать дальше, не придумывать решение
- README устаревает при миграции стека — частая находка
