# Журнал противоречий

Список несоответствий и расхождений в проекте: между JSON-данными и layouts, между RU и EN, между источниками правды, мёртвых полей и пр. Заполняется при аудите.

**Логика:**
- Раздел **«Открытые»** — что нужно сделать. Закрытые сразу удаляются (история — в git log).
- Раздел **«Намеренно оставлено»** — что выглядит как противоречие, но на самом деле не баг (требование заказчика, ограничение языка и т.п.). Чтобы при следующем аудите не натыкаться повторно.

**Формат записи:**
- **Место** — файл и строка
- **Что** — цитаты обеих сторон
- **Почему противоречие / Почему оставлено** — по существу

## Открытые

### 1. Биография и trust facts захардкожены в about.html
- **Место:** `layouts/_default/about.html:34-51, 62-73`
- **Что:** В шаблоне хардкодом сидят:
  - ФИО основателя в заголовке (именительный): `"Грозный Эдуард Николаевич"` / `"Eduard Nikolaevich Grozny"` — :34
  - Биография: `"Основатель и руководитель детективного агентства. Офицер в отставке — старший оперуполномоченный по особо важным делам. Ветеран боевых действий, награждён государственными наградами."` (и EN) — :37
  - Список из 3 преимуществ: «{years} лет детективной деятельности / Лицензия МВД РФ / Всесторонняя компетентность» — :40-49
  - Trust facts (большие цифры): `{years}` + «лет работы», «РФ+ / по всей стране и за рубежом», `100%` + «конфиденциальность» — :62-73
- **Почему противоречие:** Контент страницы должен лежать в `data/pages/about.json`, шаблон должен только рендерить. Сейчас же изменение биографии или списка требует править Go-шаблон, что должен делать только разработчик. Также `about.json` уже частично использует структуру (timeline, expertise, warning), но founder и trust facts выпадают.
- **Решение (черновое):** добавить в `about.json` блоки `founder.bio`, `founder.credentials` (массив с шаблонами `{years} {years_word} ...`), `trustFacts` (массив объектов `{ value, label }`). В шаблоне — `range` и `replace` плейсхолдеров.

### 2. Хардкод текста в list.html и single.html (UI-метки и тексты)
- **Место:** `layouts/_default/list.html` (header, badge «Дело», «Читать», «Наверх», emptyState), `layouts/_default/single.html` (бейдж «Реальное дело из практики», «мин/слов», «Продолжить чтение», «Предыдущая/Следующая», CTA-блок «Нужна помощь детектива?»)
- **Что:** Десятки строк двуязычных меток через `{{ if eq $lang "ru" }}…{{ else }}…{{ end }}` прямо в шаблонах статей и их списка.
- **Почему противоречие:** Те же страницы (home, price, contact, about) уже забирают тексты из `data/pages/*.json`. Шаблоны статей — нет. При смене формулировок UI требуется править Go-шаблон.
- **Решение (черновое):** добавить `data/pages/blog.json` с блоками `header`, `badge`, `meta` (мин/слов), `read`, `back`, `empty`, `prevNext`, `cta` (двуязычные строки). Подключить как `$p := hugo.Data.pages.blog` в обоих шаблонах. Заодно вынести склонение «статья/статьи/статей» (`list.html:74`) в новый partial по образцу `years-label.html`.

## Намеренно оставлено

### Год 2031 в футере
- **Место:** `layouts/partials/footer.html:69` — `© {{ hugo.Data.company.founded }}–2031`
- **Почему:** требование ТЗ заказчика, не трогать. Зафиксировано в CLAUDE.md → «Год в футере».

### Творительный падеж «Грозным Эдуардом Николаевичем» хардкодом
- **Место:** `layouts/_default/about.html:37` — внутри биографии
- **Почему:** русский язык требует склонения по контексту, плодить отдельные поля для каждого падежа в `company.json` — перебор. Биография целиком всё равно сидит в шаблоне (см. п.1 «Открытые») — когда вынесем её в JSON, склонение естественно уйдёт туда же.

### Title «Прайс» vs h1 «Услуги» на странице /price
- **Место:** `layouts/_default/price.html` (h1) и meta-title
- **Почему:** «Услуги» — требование закона о рекламе, `/price` в URL уже проиндексирован поисковиками. Зафиксировано в CLAUDE.md → «Страница price».

### Префикс «г.» перед городом только в RU
- **Место:** `layouts/_default/contact.html:49`, `layouts/partials/footer.html` — `{{ $cityPrefix }}{{ index $c.address.city $lang }}` (для RU `"г. "`, для EN пусто)
- **Почему:** в EN-вариантах адресов префикс «City of» не используется, в RU «г. Москва» — деловой стандарт. Условный префикс по языку — нормальная локализация, не хардкод данных.

### Inline SVG в hero badges на главной
- **Место:** `layouts/index.html:55-59` — три SVG бейджей под заголовком (галочка-в-круге, печать, метка-точки)
- **Почему:** filled-стиль (`fill="currentColor"`), несовместим с outlined `partials/icons/*.html` (lucide-style). Замена на partial поломала бы визуал. Если когда-то добавим filled-варианты иконок в partials — можно унифицировать.
