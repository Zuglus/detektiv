## 2. Дизайн-система

### 2.1 Цветовая палитра

```css
:root {
  /* Основные */
  --color-dark:      #1a2e1f;   /* фон тёмных секций, герои */
  --color-primary:   #2d6a4f;   /* основной зелёный */
  --color-accent:    #52b788;   /* акцентный зелёный, CTA primary, иконки */

  /* Нейтральные */
  --color-bg:        #f5f0e8;   /* фон светлых секций (тёплый белый) */
  --color-white:     #ffffff;
  --color-text:      #1a1a1a;   /* основной текст */
  --color-text-sec:  #5a5a5a;   /* вторичный текст, подписи */
  --color-border:    #ddd8cf;   /* границы карточек */

  /* Статусные */
  --color-gold:      #c9a84c;   /* лицензия, статус, детали доверия */
  --color-warning:   #f4a261;   /* блок "Остерегайтесь мошенников" */
}
```

**Запрещено:** вводить оранжевый (#FF6B35) как акцентный цвет CTA — это разрушает зелёную идентичность бренда. Янтарный (`--color-warning`) используется только для предупреждающих блоков.

### 2.2 Типографика

```css
/* Заголовки — авторитет, история */
--font-display: 'Cormorant Garamond', serif;

/* Интерфейс, тело — читаемость */
--font-body: 'Manrope', sans-serif;
```

**Загрузка шрифтов — только необходимые начертания:**

```html
<link href="https://fonts.googleapis.com/css2?family=Cormorant+Garamond:wght@400;600;700&family=Manrope:wght@400;500;600&display=swap" rel="stylesheet">
```

Без италиков, без 300 и 800 — экономия ~40KB к критическому пути.

**Шкала размеров:**

| Уровень | Размер | Шрифт | Вес | Применение |
|---------|--------|-------|-----|------------|
| Display | 56–64px | Cormorant Garamond | 700 | Герой главной |
| H1 | 40px | Cormorant Garamond | 700 | Заголовок страницы |
| H2 | 28px | Cormorant Garamond | 600 | Заголовок секции |
| H3 | 20px | Manrope | 600 | Заголовок карточки |
| Body | 16px | Manrope | 400 | Основной текст |
| Small | 14px | Manrope | 400 | Подписи, метки |
| Caption | 12px | Manrope | 500 | Теги, лейблы (uppercase + letter-spacing: 2px) |

**Межстрочный интервал:**
- Длинные тексты (статьи, описания): `line-height: 1.6`
- Заголовки: `line-height: 1.2`
- UI элементы: `line-height: 1.4`

### 2.3 Система отступов

Базовая единица: **8px**

```
4px   — минимальный (иконка + текст)
8px   — между строками внутри элемента
16px  — внутренний отступ карточки (padding small)
24px  — между элементами в блоке
32px  — внутренний отступ карточки (padding normal)
48px  — между карточками и секциями
64px  — padding секции (мобайл)
96px  — padding секции (десктоп)
```

### 2.4 Кнопки — иерархия

Три уровня. На одном экране может быть только **одна primary**.

```
Primary:   фон --color-accent, белый текст, border-radius: 6px, padding: 14px 28px
Secondary: прозрачный фон, рамка --color-accent, текст --color-accent, те же размеры
Ghost:     только текст --color-accent, без рамки и фона, padding: 8px 0
```

**Hover-состояния:**
- Primary: `background: #3d9a6a` (на 15% светлее)
- Secondary: `background: rgba(82,183,136,0.08)`
- Ghost: `text-decoration: underline`

**Правило:** все интерактивные элементы минимум `44px` по высоте (доступность).

### 2.5 Карточки

```css
.card {
  background: var(--color-white);
  border: 1px solid var(--color-border);
  border-radius: 12px;
  padding: 24px 28px;
  /* Выравнивание по высоте — всегда stretch */
  align-self: stretch;
}

.card:hover {
  transform: translateY(-3px);
  box-shadow: 0 8px 24px rgba(0,0,0,0.08);
  transition: all 0.2s ease;
}
```

Карточки в сетке — всегда `align-items: stretch`, чтобы не было «рваного» нижнего края.

## 5. Прайс

### 5.1 Ключевое противоречие

Страница называется **«Прозрачные цены»**, но цен нет. Это подрывает доверие.

**Решение:** добавить диапазоны цен в каждую карточку услуги.

Формат: `от XX 000 ₽` или `XX 000 – XX 000 ₽`

Если конкретные цены нежелательны — переименовать страницу в **«Услуги»** и убрать слово «прозрачные» из заголовка.

### 5.2 Иерархия кнопок

Текущее: 10 одинаковых primary-кнопок «Получить консультацию» подряд.

**Новое:**
- Внутри карточки: `Secondary` кнопка
- Внизу страницы: одна `Primary` кнопка «Получить консультацию»

### 5.3 Навигация по категориям — якорные ссылки

Sticky-навигация требует JS — несовместимо со статическим сайтом.

**Решение:** блок якорных ссылок в начале страницы:

```html
<nav class="price-nav">
  <a href="#basic">Базовые услуги</a>
  <a href="#surveillance">Наблюдение</a>
  <a href="#corporate">Корпоративные</a>
  <a href="#complex">Сложные</a>
  <a href="#search">Розыск</a>
</nav>
```

Плюс кнопка «↑ Наверх» в конце каждой секции. Нет JS — нет багов на мобильных.

### 5.4 Блок «Профессиональный результат требует инвестиций»

Текущий текст абстрактный и воспринимается как оправдание высоких цен без конкретики. **Убрать или переписать** с конкретными примерами: что входит в стоимость, почему цена формируется именно так.

## 8. Статьи — список

### 8.1 Рубрики — статическое решение

27 статей без структуры. JS-фильтры несовместимы со статическим сайтом.

**Вариант А — рубрики как секции на одной странице:**

```
## Семейные дела
[карточки]

## Бизнес и корпоративные
[карточки]

## Поиск людей
[карточки]
```

Якорные ссылки вверху страницы переходят к нужной секции.

**Вариант Б — отдельные HTML-страницы по рубрикам:**

```
/articles/index.html       — все статьи
/articles/family.html      — семейные дела
/articles/business.html    — бизнес
/articles/search.html      — поиск людей
```

Рубрики вверху как обычные ссылки. Без JS, без багов на мобильных.

Рекомендуется Вариант А если статьи редко обновляются, Вариант Б если планируется рост контента.

### 8.2 Карточки статей

Текущее: все карточки одинаковые, тег «Кейс» теряется.

**Первая статья на странице** — featured (широкая, на всю строку или 2/3 ширины).

Добавить дату публикации под тегом.

Тег «Кейс» — сделать ярче: фон `--color-accent` 20% прозрачность.

### 8.3 Миниатюры

Если есть возможность добавлять изображения — добавить тематические монохромные фото или иллюстрации к карточкам. Если нет — оставить текстовые карточки, они работают.

## 9. Статья — одна

### 9.1 Ширина текстового блока — КРИТИЧНО

Текущая ширина текста ~1200px. Это физически утомляет глаза.

```css
.article-content {
  max-width: 720px;    /* золотой стандарт для длинного чтения */
  margin: 0 auto;
  line-height: 1.6;
  font-size: 16px;
}
```

### 9.2 Типографика внутри статьи

- Абзацы: `margin-bottom: 20px`
- Нет H2/H3 в тексте — добавить структурирование длинных статей заголовками
- Цитаты или выделения (если есть) — `border-left: 3px solid --color-accent`, `padding-left: 20px`

## 12. Мобильная версия

### 12.1 Липкая панель снизу

```
┌──────────────────────────────────────┐
│  📞 Позвонить    💬 Написать         │  ← fixed bottom, z-index: 100
└──────────────────────────────────────┘
```

- «Позвонить» — `href="tel:+79150010025"`, кнопка Primary
- «Написать» — ссылка на Telegram или WhatsApp, кнопка Secondary
- Высота панели: 56px, фон `--color-white`, тень сверху
- Скрывать при скролле вниз, показывать при скролле вверх (или показывать постоянно)

### 12.2 Размеры

- Базовый шрифт: минимум **16px**
- Все тач-зоны: минимум **44×44px**
- Отступы между карточками: **24px**

### 12.3 Главная на мобильном

- Показывать 4 услуги, остальные скрыть под «Все услуги →»
- Карточки — одна в строку
- Герой: только заголовок + кнопка «Позвонить» крупно

### 12.4 Навигация

Гамбургер-меню. При открытии — полноэкранный оверлей с крупными пунктами (`font-size: 24px`).

### 12.5 Производительность

Тёмно-зелёные градиентные блоки — проверить на мобильных устройствах скорость рендеринга при скролле. Если есть «подтормаживание» — упростить до сплошного цвета без градиента.

## 13. SEO и разметка

### 13.1 Schema.org — LocalBusiness

Добавить в `<head>` главной страницы. Лицензия МВД — сильный trust signal для поисковиков.

```html
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "LocalBusiness",
  "name": "Детективное агентство",
  "description": "Лицензированное частное детективное агентство. Москва, с 2010 года.",
  "telephone": "+79150010025",
  "address": {
    "@type": "PostalAddress",
    "streetAddress": "Проспект Мира, 27",
    "addressLocality": "Москва",
    "addressCountry": "RU"
  },
  "foundingDate": "2010",
  "license": "МВД РФ 50ЧД2021000323"
}
</script>
```

### 13.2 Open Graph — для статей

На каждой странице статьи добавить метатеги для корректного отображения при шаринге в мессенджерах и соцсетях:

```html
<meta property="og:title" content="Любовь по переписке — Детективное агентство">
<meta property="og:description" content="Реальный кейс: как мы помогли гражданину США...">
<meta property="og:type" content="article">
<meta property="og:url" content="https://домен/articles/love-by-mail.html">
```

Если нет изображений для статей — `og:image` можно поставить логотип агентства.
